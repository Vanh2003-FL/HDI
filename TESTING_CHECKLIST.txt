## Pre-Testing

□ Backup database
  □ pg_dump your_database > backup_$(date +%Y%m%d).sql
  □ Verify backup file size is reasonable
  
□ Check Odoo service status
  □ sudo systemctl status odoo
  
□ Review log file location
  □ Confirm path: /var/log/odoo/odoo.log (or your configured path)

## Module Installation/Update

□ Stop Odoo service
  □ sudo systemctl stop odoo
  
□ Update ngsd_base module
  □ ./odoo-bin -c config.conf -d database_name -u ngsd_base
  
  Watch for:
  □ ✅ No ImportError messages
  □ ✅ No XMLSyntaxError messages  
  □ ✅ No ModuleNotFoundError
  □ ✅ Module upgrade completes successfully

□ Check logs during update
  □ tail -f /var/log/odoo/odoo.log
  
  Expected:
  □ ✅ "Registry loaded" message appears
  □ ✅ No Python exceptions
  □ ⚠️  Warnings are OK (states parameter, duplicate labels)

## Error Scenarios to Check

□ **Python Errors to Watch For**
  ❌ ImportError: cannot import name 'Domain'
  ❌ ModuleNotFoundError: No module named...
  ❌ SyntaxError: invalid syntax
  ❌ IndentationError
  ❌ AttributeError on model methods

□ **XML Errors to Watch For**
  ❌ XMLSyntaxError: Opening and ending tag mismatch
  ❌ ParseError: not well-formed
  ❌ Attribute 'xyz' redefined
  ❌ View inheritance failed

□ **Runtime Errors to Watch For**
  ❌ Unknown parameter 'states' (if still appearing)
  ❌ Field 'xyz' does not exist
  ❌ Can't find model abc.xyz

## Rollback Plan (If Needed)

□ **If Critical Issues Found**
  1. Stop Odoo: sudo systemctl stop odoo
  2. Restore database: psql database_name < backup_YYYYMMDD.sql
  3. Revert code: git reset --hard HEAD~1 (or restore from backup)
  4. Restart Odoo: sudo systemctl start odoo
  5. Document issues for fixing
